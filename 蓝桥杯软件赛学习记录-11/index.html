也是很久没有更新过了，原因有很多，一个是因为我开学了。新学期的课程比上个学期多了好多倍，基本天天满课，甚至周末还有课。而且大部分还是数学、物理之类的这种我不太擅长的课。再加上算法也学到了数学方面的还有动态规划这些比较难的地方了，本来时间就不充裕，速度自然也慢下来了。

不过剩下的时间不多了，还有一个月左右就要正式比赛了，还是先要把重心放在算法学习上

数学方面的一些知识是很重要的，先从进制转换开始

进制转换

10进制转k进制

简单来说就是给你一个10进制的数，把它转为任意进制

这个不难，只要用这个数对进制取模，然后再除以进制，不断重复，最后再把结果反转一下，就能得出答案

k进制转10进制

跟上面那题相反，给一个任意进制的数，将它转为10进制

这个也比较好想，从末位开始乘进制的n次方就行了

二进制枚举

简单来说就是利用二进制数的特性来高效枚举所有可能的组合，可以用在排列组合问题上

效率要比递归搜索高，但数据范围不能太大

最大公因数

c++里有个自带的函数可以求最大公因数， __gcd(a, b) ，gcd前面是两个下划线，而且变量a和b数据类型必须相同

也可以自己写一个

可以用欧几里得算法，也叫辗转相除法。代码我还看不懂，不过也不长，直接记下来也不难

(好吧，我突然看懂了=_=)

最小公倍数

可以直接暴力判断，我以前就是这么干的，不过还有效率更高的方法

首先要知道一个公式 最大公因数 x 最小公倍数 = 两数相乘

gcd(a,b) x lcm(a,b) = a x b

所以

lcm(a,b) = a x b / gcd(a,b)

那就可以先通过上面的欧几里得算法求出最大公因数，然后直接套公式

快速幂

用来求a的b次方模p

当数据量比较大的时候就需要用到快速幂算法来解决

质数

质数判断

可以直接全部遍历一遍判断，也可以简单做个优化，只需要判断到根号n就可以

质数筛

我只学了简单的埃氏筛，线性筛没看懂，不过这个应该就够用了
